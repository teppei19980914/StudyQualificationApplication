## 1. プロジェクト概要

### 1.1 目的

社会人の自己学習者が、複数の学習目標を計画的に管理し、学習の習慣化と継続を支援するデスクトップアプリケーションを提供する。

### 1.2 対象ユーザー

仕事をしながら資格取得やスキルアップの勉強を行う社会人。帰宅後の限られた時間で計画的に学習を進めたいユーザー。

### 1.3 アプリケーション名

Study Planner（Study Qualification Application）

## 3. 非機能要件

### NFR-01: 性能

| ID | 要件 | 基準 |
|----|------|------|
| NFR-01-01 | アクティビティチャートの期間切替が即座に完了する | 再計算なし（事前計算済みデータの切替） |
| NFR-01-02 | 祝日判定が高頻度呼び出しに対応する | 年単位キャッシュ |
| NFR-01-03 | ダッシュボードのrefreshが体感で遅延なく完了する | 全計算を1回のrefreshで完了 |

### NFR-02: 信頼性

| ID | 要件 | 基準 |
|----|------|------|
| NFR-02-01 | データが正しく永続化される | JSON形式、UTF-8エンコーディング |
| NFR-02-02 | JSONパースエラー時にアプリがクラッシュしない | 空リスト返却でグレースフルデグレード |
| NFR-02-03 | インポート時にJSONバリデーションが行われる | 不正JSONはValueErrorで拒否 |
| NFR-02-04 | 実績通知が重複生成されない | dedup_keyによる一意性保証 |
| NFR-02-05 | カスケード削除が正しく実行される | 目標→タスク、書籍→タスクの連鎖 |

### NFR-03: 保守性

| ID | 要件 | 基準 |
|----|------|------|
| NFR-03-01 | 3層アーキテクチャ（GUI/Service/Repository）に従う | GUI層にビジネスロジックを含まない |
| NFR-03-02 | 全コードに型ヒントを付与する | mypy strictモード対応 |
| NFR-03-03 | ruffによるリント・フォーマットに適合する | エラー0件 |
| NFR-03-04 | テストカバレッジ100%を目指す | 新規コードは必須 |
| NFR-03-05 | 全モジュールにロギングを実装する | logging標準ライブラリ使用 |
| NFR-03-06 | ログはファイルに永続化される | logs/ディレクトリ、ローテーション付き |

### NFR-04: 使用性

| ID | 要件 | 基準 |
|----|------|------|
| NFR-04-01 | ダーク/ライトテーマを提供する | Catppuccinパレット |
| NFR-04-02 | テーマ切替が即座に反映される | 全ウィジェットのQSS更新 |
| NFR-04-03 | ダッシュボードが自由にカスタマイズできる | ドラッグ&ドロップ、追加/削除/リサイズ |
| NFR-04-04 | 日本語UIを提供する | 全ラベル・メッセージ日本語 |
| NFR-04-05 | 破壊的操作には確認ダイアログを表示する | 目標削除、データインポート、全データ削除 |
| NFR-04-06 | ナビゲーションがアニメーション付きで直感的に操作できる | スライドイン/アウト |

### NFR-05: 拡張性

| ID | 要件 | 基準 |
|----|------|------|
| NFR-05-01 | 新しいダッシュボードウィジェットを容易に追加できる | WIDGET_REGISTRYへの登録のみ |
| NFR-05-02 | 新しいページを容易に追加できる | QStackedWidgetへの追加 |

| NFR-05-04 | 実績通知の閾値を容易に追加できる | 閾値リストへの値追加のみ |

### NFR-06: 環境

| ID | 要件 | 基準 |
|----|------|------|
| NFR-06-01 | Python 3.12以上で動作する | 3.12+ |
| NFR-06-02 | Windows環境で動作する | Windows 11 |
| NFR-06-03 | PySide6 6.7.0以上を使用する | Qt6 |
| NFR-06-04 | 外部サーバーへの接続を必要としない | スタンドアロン動作 |

## 4. データ要件

### DR-01: データモデル

| ID | 要件 |
|----|------|
| DR-01-01 | 目標データ（Goal）: id, why, when_target, when_type, what, how, color, created_at, updated_at |
| DR-01-02 | タスクデータ（Task）: id, goal_id, title, start_date, end_date, status, progress, memo, book_id, order, created_at, updated_at |
| DR-01-03 | 学習ログデータ（StudyLog）: id, task_id, study_date, duration_minutes, memo, task_name, created_at |
| DR-01-04 | 書籍データ（Book）: id, title, status, summary, impressions, completed_date, start_date, end_date, progress, created_at, updated_at |
| DR-01-05 | 通知データ（Notification）: id, notification_type, title, message, is_read, dedup_key, created_at |

### DR-02: 永続化

| ID | 要件 |
|----|------|
| DR-02-01 | 全データをJSONファイルで永続化する |
| DR-02-02 | データファイルはdata/ディレクトリに保存する |
| DR-02-03 | 文字エンコーディングはUTF-8を使用する |
| DR-02-04 | JSONフォーマットはインデント2のpretty printとする |
| DR-02-05 | ディレクトリが存在しない場合は自動作成する |

### DR-03: バリデーション

| ID | 要件 |
|----|------|
| DR-03-01 | 目標の全フィールド（why/when/what/how）は空不可 |
| DR-03-02 | タスクの進捗率は0-100の範囲 |
| DR-03-03 | タスクの開始日は終了日以前 |
| DR-03-04 | 学習ログの学習時間は1分以上 |
| DR-03-05 | 書籍のタイトルは空不可 |
| DR-03-06 | 書籍の進捗率は0-100の範囲 |
| DR-03-07 | 書籍の開始日は終了日以前 |

### DR-04: データ管理

| ID | 要件 |
|----|------|
| DR-04-01 | データをZIPファイルにエクスポートできる（6ファイル対象） |
| DR-04-02 | ZIPファイルからデータをインポートできる（JSONバリデーション付き） |
| DR-04-03 | 全データを削除できる（settings.jsonは保持） |
| DR-04-04 | インポート時に不明なファイルは無視する |
| DR-04-05 | settings.jsonは複数サービスで共有読み書きする（他キー保持） |

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| 3W1H | Why（なぜ）、When（いつまでに）、What（何を）、How（どうやって） — 目標登録のフレームワーク |
| ストリーク | 連続して学習を行った日数。今日未学習でも昨日までの連続は維持 |
| 実績 | 累計学習時間、学習日数、連続学習日数が一定閾値に到達した際の達成記録 |
| dedup_key | 重複防止キー。通知の一意性を保証する文字列（例: "total_hours:100"） |
| BOOK_GANTT_GOAL_ID | 書籍タスクを識別する特殊goal_id値（"__books__"） |
| column_span | ダッシュボードウィジェットの幅。1=半幅、2=全幅 |
| QSS | Qt Style Sheet。QtウィジェットのCSS風スタイル記述 |
| Catppuccin | カラーパレット名。Mocha（ダーク）/Latte（ライト）を使用 |
